Git操作流程

一、先前准备

配置全局变量

git config --global user.name "lalo"

git config --global user.email "814112150@qq.com"

生成的文件地址

C:\Users\Lalo\.gitconfig

二、提交操作

找一个文件夹作为仓库，初始化版本库

git init

将文件添加到临时暂存区

git add .

提交文件

git commit-m "提交提示信息"

三、版本操作

查看git仓库状态（是否有改动需要上传）

git status

提交日志

git log --pretty=oneline

回退到上一次提交

git reset --hard HEAD^1

git reset --soft HEAD^1

HEAD是一个指针，指向最新版本

版本穿越

git reflog

穿越到任意版本

git reset --hard id号

从暂存区中删除文件

git rm xxx --cached

将文件还原到上个分支前

git checkout *

四、分支操作

创建分支

git branch 分支名

查看分支

git branch -v

切换分支

git checkout 分支名

将当前分支与目标分支合并

git merge 目标分支

合并分支的操作不影响分支的数据

分支冲突

查看冲突

git diff

解决：手动

然后重新add commit

五、代码托管提交到码云 *

情况一：本地git库进行提交

其他问题

git clone和git pull的区别

git clone是用来克隆远程地址的整个git库，本地不需要有git仓库，git pull是将本地库与远程库进行同步，也可以将远程分支与本地仓库进行merge。

supplier开发文档编写

首页：

1、查询所有供应商

入参：year String 年份

SELECT supplier,SUM(money) FROM ods_supplier_ys WHERE YEAR = ‘2020’ GROUP BY supplier
ORDER BY SUM(money) DESC

2、根据供应商名字查工厂

入参：year String 年份 ，s_name

SELECT ofy.factory,SUM(ofy.money) FROM supplier_factory sf ,ods_factory_ys ofy WHERE sf.s_name=’福州福森’ AND ofy.factory=sf.f_name
AND ofy.year=’2020’ GROUP BY ofy.factory ORDER BY SUM(ofy.money) DESC;

3、根据工厂名字查货号

SELECT goods_sku,SUM(money) FROM ods_factory_ys WHERE factory=’寿光佳泰’
AND YEAR=’2020’ GROUP BY goods_sku ORDER BY SUM(money) DESC;
